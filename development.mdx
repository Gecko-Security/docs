---

<Info>
  **Prerequisites**
  - Node.js `v20.17.0` or later
  - Git
  - This repo checked out locally
</Info>

Use Mintlify's CLI to preview the docs, check your changes, and catch broken
links before you push.

<Steps>
<Step title="Install the Mintlify CLI">

```bash
npm i -g mint
```

If you do not want a global install, use `npx mint dev` for a one-off preview.
</Step>

<Step title="Start the local preview">

From the repo root, where `docs.json` lives, run:

```bash
mint dev
```

Open `http://localhost:3000` in your browser. Mintlify reloads the preview when
you save changes.
</Step>

<Step title="Validate links before you ship">

```bash
mint broken-links
```

Run this after you add or rename pages so you catch bad internal links early.
</Step>
</Steps>

## Useful commands

```bash
mint dev --port 3333
mint update
npx mint dev
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 is already in use">
    Run `mint dev --port 3333`. If you keep `3000`, Mintlify will try the next
    open port automatically.
  </Accordion>

  <Accordion title="The local preview looks older than production">
    Run `mint update`. If your local CLI is too old to support that command,
    reinstall it with `npm i -g mint@latest`.
  </Accordion>

  <Accordion title='`mint version` shows `none`'>
    Your network may be blocking Mintlify's client download. Ask your IT team
    to allow `releases.mintlify.com`.
  </Accordion>
</AccordionGroup>
