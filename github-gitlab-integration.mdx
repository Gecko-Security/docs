## Connect source control first

Use **Settings** &gt; **Git Settings** to connect GitHub or GitLab.

This is the highest-value integration for most teams because it makes the rest
of the Gecko workflow easier:

- repositories show up directly in the scan flow
- scans are easier to repeat
- remediation stays tied to real code ownership

<Tip>
  Start with the provider your team already uses for private code. Use public
  OSS URLs only when you are evaluating Gecko on public repositories.
</Tip>
