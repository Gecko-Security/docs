Gecko works best when you use it as a repeatable workflow:

1. Connect a source of code.
2. Run a scan.
3. Review the findings.
4. Fix what matters.
5. Rescan and confirm the result.

That loop is the product. The rest of Gecko exists to make that loop faster,
clearer, and easier to scale across a team.

## What Gecko scans

Gecko analyzes real code paths in repositories and code snapshots to surface
security issues that deserve engineering attention.

For most teams, that means:

- starting with a baseline scan on the default branch
- reviewing the highest-signal findings first
- turning validated issues into remediation work
- rerunning scans after fixes land

## Recommended first scan

<Steps>
<Step title="Start with one repository">
  Pick a repository the team understands well enough to validate quickly.
</Step>
<Step title="Use the default branch first">
  A baseline scan gives you the cleanest picture of the current security state
  before you layer in more workflow complexity.
</Step>
<Step title="Review the first findings with a code owner">
  Use the first result set to calibrate risk, validate exploitability, and
  align on how the team will fix issues going forward.
</Step>
<Step title="Rescan after remediation">
  The rescan is what proves the repository is in a better state than before.
</Step>
</Steps>

## What to expect

<AccordionGroup>
  <Accordion title="The first scan is a baseline, not a complete program">
    The first result set helps your team understand what Gecko is surfacing and
    how to route the work.
  </Accordion>
  <Accordion title="Some findings are immediate fixes and some require validation">
    Gecko gives code context and remediation guidance, but your team still owns
    the final call on business impact and exploitability.
  </Accordion>
  <Accordion title="The workflow gets stronger after the first remediation cycle">
    Once you have seen one issue move from discovery to verified fix, the rest
    of the platform becomes much easier to adopt.
  </Accordion>
</AccordionGroup>
