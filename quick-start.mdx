---
title: "Quick start"
description: "Connect a repository, run your first scan, and review findings."
---

<Steps>
<Step title="Connect a repository">
  Go to the dashboard and connect GitHub or GitLab via **Settings > Git Settings**.

  If you're evaluating Gecko on public code, paste a public repository URL instead.
</Step>

<Step title="Run a baseline scan">
  Pick a repository your team knows well. Run a scan on the default branch.

  This gives you the current security picture before layering in more workflow.
</Step>

<Step title="Review findings">
  Start with anything touching auth, secrets, remote execution, or external network access.

  Each finding includes code context, explanation, and remediation guidance.
</Step>

<Step title="Fix and rescan">
  Apply a fix, then rerun the scan to confirm the issue is resolved.
</Step>
</Steps>

After your first scan loop works, add [integrations](/integrations), [team roles](/teams-permissions), and [SSO](/access/okta-saml-onboarding).
